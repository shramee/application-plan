Taxonomies

Data that can be used to open modules object for same type, for example, for People module, Income Group, Internet Access, City etc. may need to be set as Taxonomy, so that users can get People from a particular City or Income group.

Meta data

This is data that is stored in database but is not sortable by, but can be searched for. For example Contact Number, Email etc. would be stored in db but one won't need stuff like, getting all People with Contact number 9898989898, Coz there is likely to be only one (or from the same family), they may instead want to search for that person.

Search/Archive pages

This info will be shown on pages with multiple entries, like search pages and Taxonomy term pages ( Like on page containing all the People living in Kerala ).

Left box contain available data fields, drag and sort them in the yellow box on the right.

Display on details page

This info will be shown on page with single entry, for example the page for Alex Johnson.
